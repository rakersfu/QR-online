Nginx Alpine 构建流程
1. 基础镜像
Alpine Linux → 极简系统，体积小，启动快。

2. 用户与权限
创建 nginx 用户和组（UID/GID=101）。

避免以 root 运行，提高安全性。

3. 安装 Nginx
官方支持架构（x86_64、arm64）：

从 nginx.org 下载预编译二进制包。

校验 RSA 签名和 SHA512 校验和。

非官方架构：

下载 pkg-oss 源码。

本地编译生成二进制文件。

4. 依赖管理
安装必要依赖：openssl、pcre2、zlib 等。

保证 Nginx 在 Alpine 环境下正常运行。

5. 额外工具
安装 gettext-envsubst → 用于模板变量替换。

安装 tzdata → 支持时区设置。

6. 日志处理
将访问日志重定向到 stdout。

将错误日志重定向到 stderr。

方便 Docker 日志系统收集。

7. 启动入口
使用 docker-entrypoint.sh 脚本作为入口点。

默认命令：

bash
CMD ["nginx", "-g", "daemon off;"]
保持 Nginx 在前台运行，符合容器化要求。

✅ 总结
官方支持架构 → 使用官方编译好的二进制。

非官方架构 → 构建时自动下载源码并编译。

最终镜像 → 包含 Nginx 二进制 + Alpine Linux，轻量、安全、适合生产环境。
